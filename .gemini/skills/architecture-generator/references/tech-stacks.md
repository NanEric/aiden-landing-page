# Technology Stack Options

Guidelines for choosing technology stacks based on project types.

## 1. Modern Web Applications (Full Stack)
- **Frontend**: React (Next.js), Vue (Nuxt.js), or Svelte (SvelteKit).
- **Backend**: Node.js (Express/FastAPI), Python (Django/FastAPI), or Go (Gin/Echo).
- **Database**: PostgreSQL (Relational), MongoDB (NoSQL), or Redis (Caching).
- **Deployment**: Vercel, Netlify, Docker + Kubernetes, or AWS/GCP.

## 2. Real-time Systems
- **Protocol**: WebSockets, gRPC, or MQTT.
- **Message Broker**: RabbitMQ, Kafka, or Redis Pub/Sub.
- **Language**: Go, Erlang (Elixir), or Rust.

## 3. Microservices Architecture
- **Service Mesh**: Istio or Linkerd.
- **API Gateway**: Kong, Nginx, or Traefik.
- **Observability**: Prometheus, Grafana, Jaeger, or VictoriaMetrics.

## 4. Mobile Applications
- **Cross-Platform**: Flutter, React Native, or Compose Multiplatform.
- **Native**: Swift (iOS), Kotlin (Android).

## Selection Criteria
- **Scalability**: Can it handle growth?
- **Developer Velocity**: How fast can the team build?
- **Ecosystem**: Are there libraries and support?
- **Cost**: Infrastructure and licensing costs.
